CloudBees mobile services

This project showcases CloudBees mobile services. We have assembled several template projects that you can refer to (and clone/build from) that show the full range of CloudBees mobile services, from building mobile apps, deploying server apps, integrating data, push messages - all hosted, by CloudBees.

The demo app

The best way to learn about mobile app development and how to use CI in the mobile space is by using a real, working example. We created an example that you're probably familiar with - a restaurant review app. We call it Gasp, a bit of a variation on Yelp. Users create restaurant reviews, and users read reviews others have made near their location. The design of this app is shown in this clickable diagram:

While the app is simple in concept, the code addresses standard problems that a lot of mobile apps have to deal with. For example:

  • Who's using your app? You need to keep this information in a database on the backend, of course. The kind of data you store on a per-user basis will be quite application-specific, and you probably need to track user information in other systems outside of the app-specific database.
  • How do you let your users know when something interesting happens that is relevant to them and their location? You need to make use of a "notification service" and a "location service", along with some application logic on the backend. You need to have a way to broadcast the notification, but only to specific users based on their location or information that you have on the backend.
  • How do you use compact "something happened" style notifications, but synchronize the specific data onto the device needed by your app?
  • How do you test properly when you have both Android and iOS front-ends connected to back-end logic? How should you factor your code to make things more testable and to enable multi-stream development by different teams proceeding at different paces?
  • How do you easily integrate and synchronize data between the multiple databases and hosted services you use within your mobile application and tie them to your other business systems, like Salesforce?

Solving these problems involves a lot of moving parts and code. Gasp gets you hands-on with mobile CI using Jenkins, databases, server-side apps, iOS and Android native services, AWS native services, and WEAVE@cloud integration.
